# Plan
- Use up all the spare 2.5" hard drives
- Create a cold storage solution in addition to all pre-existing

# Constraints
- Absolute minimal spend
- In addition to no new storage (previous projects) no new compute
- Couldn't use preferred Unraid because no spare x86 compute

# Infrastructure

- Raspberry Pi 4 4GB
- mBeat 7 Port Powered USB Hub (4x USB3, 3x USB2)
- 6x Various 250GB-1TB 2.5" HDDs in external drive enclosures

# Steps
- Raspberry Pi Imager
  - Rpi OS Lite (no desktop)
- Update packages: `sudo apt update && sudo apt upgrade && sudo apt autoclean && sudo apt autoremove`
- Install relevant packages: `sudo apt install mergerfs snapraid -y`
- Identify relevant disks: `lsblk -o NAME,SIZE,TYPE,FSTYPE,UUID`
- Partition: `sudo parted /dev/sdX --script mklabel gpt mkpart primary ext4 0% 100%`
- Format :`sudo mkfs.ext4 -O ^has_journal /dev/sdX1`
- Create mountpoints: `sudo mkdir -p /mnt/disk1 /mnt/disk2 /mnt/disk3 /mnt/disk4 /mnt/array`
- Gather UUIDs: `sudo blkid`
- Add to fstab: `UUID=<uuid1>  /mnt/disk1  ext4  defaults,noatime,nofail,x-systemd.device-timeout=10  0  0
UUID=<uuid2>  /mnt/disk2  ext4  defaults,noatime,nofail,x-systemd.device-timeout=10  0  0
UUID=<uuid3>  /mnt/disk3  ext4  defaults,noatime,nofail,x-systemd.device-timeout=10  0  0
UUID=<uuid4>  /mnt/disk4  ext4  defaults,noatime,nofail,x-systemd.device-timeout=10  0  0`
  - `nofail` → don’t break boot if a disk is slow
  - `x-systemd.device-timeout=10` → wait up to 10 seconds for USB spin‑up
  - `noatime` → reduces writes on HDDs
- Add mergerfs union: `/mnt/disk* /mnt/array fuse.mergerfs defaults,allow_other,use_ino,category.create=mfs,moveonenospc=true,minfreespace=10G,nofail 0 0`
- Reload systemd and mount everything:
  - `sudo systemctl daemon-reload`
  - `sudo mount -a`
  - `df -h`
